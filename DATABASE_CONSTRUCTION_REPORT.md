**报告日期**：2026年2月

## 一、数据库构建平台与方法

### 1.1 技术架构

本数据库采用现代化Web技术栈构建，实现前后端分离架构：

- **前端技术**：HTML5 + JavaScript (ES6+) + CSS3
- **数据存储**：JSON格式结构化数据
- **版本控制**：Git/GitHub
- **部署平台**：GitHub Pages（静态站点托管）
- **自动化部署**：GitHub Actions工作流

### 1.2 数据组织方法

采用多层次数据结构设计：

```
数据库
├── 材料分类（5类）：元素、固溶体、金属间化合物、非晶、界面
├── 多温度点支持（0-1000K）
├── 多数据源支持（DFT、DPA-1、DPA-3、MD、实验）
└── 结构化JSON格式存储
```

**数据标准化**：
- 唯一ID自动生成（Alloy-XX-#####格式）
- 统一单位制（SI单位）
- 标准化命名规范
- 严格的数据验证脚本

### 1.3 数据管理工具

开发了完整的数据处理脚本集：
- `convert-data-v2.js` - CSV到JSON格式转换
- `validate-data.js` - 数据格式验证
- `check-duplicates.js` - 重复数据检测
- `update-materials.js` - 数据更新（4种模式）
